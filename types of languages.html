Языки низкого уровня

У низкоуровневых языков нет встроенных функций. Они общаются с конкретным процессором, поэтому код одного компьютера не может быть использован на другом. У каждого производителя процессоров свой собственный набор инструкций. 

Самые распространенные низкоуровневые языки — это машинный код и язык ассемблера.

Машинный код

Тот самый двоичный код — язык нулей и единиц, который компьютер понимает без дополнительной обработки. Для человека он кажется слишком сложным, и в нем легко запутаться. Поэтому в чистом виде его практически не используют. 

Язык Ассемблера

 

Сегодня язык ассемблера используют для создания программного обеспечения драйверов. Чтобы компьютер понял этот язык, все команды пропускаются через компилятор или интерпретатор и переводят обратно в двоичный код.

Языки высокого уровня

Язык ассемблера упрощает машинный код, но он все еще сложен для массового использования. Поэтому программисты начали создавать новые языки, которые больше упрощают программирование. Так появились высокоуровневые языки, синтаксис которых близок к английскому.

Понять высокоуровневый код без обработки, как и язык ассемблера, компьютер не может. Поэтому предварительно пропускает его через переводчик — компилятор и интерпретатор. 

Популярные языки программирования высокого уровня

  • Python — простой универсальный язык для новичков. Подходит для  машинного обучения, создания ПО, игр или web-сайтов. 
  • JavaScript — используется для создания мобильных и компьютерных приложений. Отличительная особенность — большая библиотека готовых плагинов и скриптов, которые существенно упрощают работу.
  • Java  — один из наиболее известных и популярных языков. Используется для создания мобильных приложений

Способ выполнения

Способ выполнения кода определяет, как исходный код преобразуется в машинные инструкции и выполняется на устройстве. Это влияет на скорость работы, переносимость между платформами и удобство разработки. Основные категории включают компилируемые, интерпретируемые и скриптовые языки, хотя границы между ними могут быть размыты (например, JIT-компиляция в интерпретируемых языках).

Компилируемые языки

  • Код компилируется в машинный язык или промежуточный байт-код до выполнения программы.

  • Высокая производительность, так как оптимизация происходит на этапе компиляции.

  • Требуют повторной компиляции для разных платформ.

Примеры языков:  C, C++, Java, C#

Интерпретируемые языки

  • Код выполняется построчно во время работы программы с помощью интерпретатора.

  • Медленнее компилируемых языков, но обеспечивают кроссплатформенность и гибкость.

  • Удобны для быстрой разработки и отладки.

Примеры языков:  Python, JavaScript, PHP

Скриптовые языки

  • Подвид интерпретируемых языков, ориентированный на автоматизацию задач и интеграцию с другими программами.

  • Часто используются для написания сценариев (скриптов) в вебе, играх или системном администрировании.

Примеры языков:  Python, Lua 

Парадигмы программирования

Парадигма программирования — это стиль написания кода, определяющий структуру программы и подход к решению задач. Большинство современных языков поддерживают несколько парадигм, но изначально проектируются с акцентом на одну из них.

Объектно-ориентированные (ООП)

  • Программа строится вокруг объектов — сущностей, объединяющих данные (поля) и методы (функции).

  • Основные принципы: инкапсуляция, наследование, полиморфизм.

 Примеры языков: Java, C#

Функциональные

  • Программа рассматривается как набор функций, которые принимают входные данные и возвращают результат.

  • Акцент на неизменяемости данных, чистоте функций (без побочных эффектов) и функциях высшего порядка.

Примеры языков:  JavaScript, Python

Процедурные

  • Программа состоит из последовательности процедур (функций), которые вызываются в определенном порядке.

  • Акцент на шаговом выполнении инструкций и изменении состояния программы.

Примеры языков:  C, Pascal

Направления разработки

В виду развития языков программирования, появлением большого количества языков высокого уровня, для более простого понимания разделения языков было принято решение разделять их по направлениям разработки.

Направления разработки в программировании — это различные области и специализации, в которых могут работать программисты и разработчики программного обеспечения. Каждое направление имеет свои особенности, технологии и инструменты. 

Основные направления разработки и их описание

  • Веб-разработка:  разработка сайтов и веб-приложений
  • Мобильная разработка: создание приложений для мобильных устройств на платформах Android и IOS
  • Настольная разработка: разработка приложений для таких операционных систем как Windows, macOS, Linux
  • Разработка игр: создание игр для разных платформ, включая ПК, консоли
  • Машинное обучение  и ИИ: создание и внедрение алгоритмов искусственного интеллекта и машинного обучения
  • Блокчейн: разработка на базе блокчейн-технологий для создания децеyтрализованных приложений (dApps)

У каждого из этих направлений разработки используются определённые языки программирования. Для более простого различия предназначения языков следует использовать такую классификацию. Подробнее о них в следующих разделах курса.
